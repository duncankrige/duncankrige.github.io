Force Plate Project – Data-Driven Athlete Performance Analysis

The Force Plate Project aimed to leverage data analytics to enhance athlete performance and optimize training in the weight room. This initiative focused on collecting, processing, and analyzing force plate readings for baseball, softball, and basketball teams, with plans to expand to other sports. Weekly data collection captured 11 key performance metrics, which were processed and visualized using R Shiny applications. These applications provided strength coaches with interactive dashboards and customizable graphs to facilitate data-driven decision-making and training adjustments.

Tools Used

  • R Shiny – For building interactive dashboards and visualizing athlete metrics
  • Microsoft Excel – For organizing and cleaning raw force plate data
  • ggplot2 (R Library) – For data visualization and trend analysis
  • Shiny Dashboard – For creating user-friendly interfaces tailored to coaching needs
  • SQL(dbeaver) – For managing structured data efficiently

Skills Learned

  • Data Analysis & Interpretation – Extracting actionable insights from force plate readings
  • Data Visualization – Developing interactive graphs to track performance trends
  • Statistical Modeling – Identifying patterns and correlations in athlete metrics
  • R Shiny Development – Building custom applications for real-time data analysis
  • User-Centered Design – Collaborating with strength coaches to ensure practical usability
  • Project Scalability – Designing adaptable tools for future expansion to additional sports

This project strengthened my ability to bridge the gap between raw performance data and real-world athletic application. By developing intuitive, data-driven tools, I enabled coaches to make informed training decisions, enhancing athlete development through a structured, analytical approach.

The Effects of Exogenous Gibberellin Exposure on Dwarf Mutant Millet Plants

This study analyzed the impact of gibberellin (a plant growth hormone) on two dwarf mutant millet variants to determine the nature of their genetic mutations. The objective was to assess whether these mutations inhibited gibberellin production or deactivated its effects through hydroxyl group binding. By examining plant growth data and response patterns, we gained insights into the underlying genetic mechanisms affecting gibberellin metabolism and its role in plant development.

Tools Used

  • R – For statistical modeling and data analysis
  • Microsoft Excel – For organizing and visualizing plant growth trends
  • Microsoft Teams – Communication
  • SharePoint – Transferring of files

Skills Learned

  • Statistical Analysis – Using R² values to evaluate model accuracy and predictability
  • Data Interpretation – Assessing plant growth trends to identify mutation effects
  • Data Visualization – Creating clear visual representations of growth responses
  • Experimental Data Analysis – Comparing treated vs. untreated samples to infer genetic impacts
  • Critical Thinking – Drawing conclusions on gibberellin metabolism based on quantitative data
  • Scientific Communication – Translating complex biological data into meaningful insights

This project strengthened my ability to work with experimental datasets, apply statistical methods, and extract actionable conclusions—valuable skills for data analysis in both scientific and business settings.

Sorting Algorithms – Performance Analysis in C++

This project explores the implementation and performance comparison of various sorting algorithms in C++, including Bubble Sort, Merge Sort, Quick Sort, Heap Sort, and System Sort. The program sorts an array of integers using each algorithm and evaluates their efficiency based on execution time and computational complexity. By analyzing the differences in sorting speeds and resource usage, the project highlights the strengths and weaknesses of each method in different scenarios.

Tools Used

  • Standard Template Library (STL) – Used for system-provided sorting functions

  • Microsoft Excel - Storing running times from files read

Solving Sudoku Puzzles – C++ Backtracking Algorithm

This project is a C++ implementation of a Sudoku solver that reads an unsolved Sudoku puzzle from a file, processes it using a recursive backtracking algorithm, and outputs the completed solution. The program follows a structured approach, ensuring that each number placement adheres to standard Sudoku rules, maintaining row, column, and 3×3 grid constraints.

Tools Used

  • C++ – Core programming language
  • Recursive Backtracking Algorithm – Systematically explores possible solutions
  • File I/O – Reads and writes Sudoku puzzles from external files
  • 2D Arrays – Stores and manipulates the Sudoku grid efficiently
  • Logical Constraint Checking – Ensures valid number placements

Skills Learned

  • Algorithm Design & Optimization – Implementing and refining backtracking for efficiency
  • Recursive Problem-Solving – Using recursion to systematically explore solutions
  • File Handling in C++ – Reading and writing structured data from files
  • Debugging & Logical Reasoning – Identifying invalid moves and improving algorithm performance
  • Data Structures (2D Arrays) – Managing and navigating grid-based data

This project strengthened my ability to develop efficient algorithms for constraint-based problem-solving, reinforcing my understanding of recursion, data handling, and structured programming in C++.
